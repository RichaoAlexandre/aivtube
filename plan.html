AI VTuber — Build Plan

A step-by-step plan to build an autonomous AI VTuber that runs live, built up from a minimal talking head to a full autonomous streamer. Each phase produces something you can run and see/hear, so you're never building blind.

Guiding principle: get a vertical slice working early (text → speech → moving mouth on screen), then add inputs, memory, and autonomy. 
live chat / voice / idle timer
   → LLM "brain" (personality + memory)
   → TTS "voice" (low-latency speech audio)
   → Live2D avatar (audio-driven lip-sync + emotion expressions)
   → OBS → Twitch / YouTube / Kick

Key insight: lip-sync is driven by the generated TTS audio, not a webcam — so no face-tracking hardware is needed.

Phase 0 Decisions & setup

Lock these before writing code; they shape everything downstream.

  1. 1Avatar type: Live2D (2D) recommended — cheaper, lighter, easiest lip-sync. (3D/VRM later for full body.)
  2. 2Brain: start with a hosted frontier LLM (Claude) for best personality with least effort; keep an Ollama local path as offline fallback.
  3. 3TTS: start with Edge-TTS (free) to prototype; upgrade to ElevenLabs Flash / Cartesia (~100ms cloud) or GPT-SoVITS / CosyVoice (open source, cloneable).
  4. 4Stack: Python orchestrator + web frontend for the Live2D renderer. Event-driven core.
  5. 5Repo hygiene: git init, virtualenv (uv), .env for keys, .gitignore for models/keys/audio.
  6. 6Voice input? For a chat-driven streamer, ASR is optional. Skip at first.
Deliverable: empty project skeleton, deps installable, keys in .env.

Phase 1 The Brain (LLM + personality)

  1. 7Pick an LLM client (Anthropic SDK for Claude / OpenAI-compatible / Ollama).
  2. 8Write a character system prompt: name, personality, speaking style, do/don'ts, output constraints (short spoken lines, no markdown, inline emotion tags like [happy]).
  3. 9Build a brain module: respond(context) → {text, emotion}. Parse emotion tags out of the reply.
  4. 10Test in a terminal REPL: type a message, get an in-character reply.
Deliverable: a CLI you can chat with that stays in character.

Phase 2 The Voice (TTS)

  1. 11Wrap a TTS backend behind a simple speak(text) → audio interface so backends are swappable.
  2. 12Stream/playback audio locally; measure latency (target: speech <1s after LLM; ideally stream TTS as the LLM streams tokens).
  3. 13(Later) Voice cloning: fine-tune GPT-SoVITS or use an ElevenLabs custom voice for a unique identity.
Deliverable: type text → hear the character speak it.

Phase 3 The Face (Live2D avatar + lip-sync)

  1. 14Get a Live2D model: buy/commission, use a free sample, or rig one in Live2D Cubism (.model3.json).
  2. 15Render it in a web frontend with pixi-live2d-display (PixiJS + Live2D Cubism SDK). Runs in browser/Electron — no VTube Studio needed.
  3. 16Audio-driven lip-sync: feed TTS audio amplitude/visemes to the model's MouthOpenY parameter.
  4. 17Expressions: map LLM emotion tags ([happy], [angry]…) to Live2D expressions/motions.
Deliverable: avatar whose mouth moves and expression changes as it speaks.

Phase 4 Wire the core loop — the vertical slice

  1. 18Stand up a backend server (FastAPI) + WebSocket to the frontend.
  2. 19Flow: input → brain → tts → {audio + emotion} → frontend plays + animates.
  3. 20Add subtitles/captions in the frontend (and/or to a text file for OBS).
  4. 21Handle interruption and a clean speaking queue (no overlapping lines).
Deliverable: end-to-end — send text, watch+hear the avatar respond. This is the MVP.

Phase 5 Inputs (live chat + optional voice)

  1. 22Live chat ingestion: connect to Twitch (IRC/EventSub) and/or YouTube Live Chat API. Or use Social Stream Ninja for 120+ platforms via webhook.
  2. 23Normalize chat events into the core loop; add selection logic (don't reply to everything — pick, batch, rate-limit).
  3. 24Moderation/safety filter on incoming chat and outgoing speech (slurs, prompt-injection, doxxing; keep on-brand).
  4. 25(Optional) Voice input via ASR (Faster-Whisper) with VAD and barge-in — for collabs/co-hosts.
Deliverable: avatar reads and responds to real live chat.

Phase 6 Memory & autonomy

  1. 26Short-term memory: rolling conversation/chat context window.
  2. 27Long-term memory: vector DB (e.g. Chroma) RAG or MemGPT-style — remember regulars, jokes, facts.
  3. 28Proactive / idle engine: when chat is quiet, generate self-initiated talk so there's never dead air.
  4. 29Director/state machine: decide when to read chat, idle-talk, or run a bit (Neuro-sama-style: separate talk loop from game loop).
Deliverable: avatar talks on its own, remembers things, never goes silent.

Phase 7 Going live (streaming)

  1. 30Install OBS Studio. Add the avatar frontend as a Browser Source; add captions; design the scene.
  2. 31Audio routing: route TTS audio into OBS (virtual audio cable / monitor) so it's captured.
  3. 32Connect OBS to Twitch / YouTube / Kick (stream key); do a private test stream.
  4. 33Add stream furniture: starting screen, BRB, alerts (follows/subs/donations) the AI can react to.
Deliverable: a working private livestream of the AI VTuber.

Phase 8 Polish & advanced features

  1. 34Singing: separate singing voice model (voice clone / RVC) triggered on request.
  2. 35Vision: "see" the game/screen via screenshots → multimodal LLM, and react.
  3. 36Mini-games / interactivity: chat commands, polls, games it can play.
  4. 37Reliability: crash recovery, watchdog, rate/cost monitoring, logging, latency budget.
  5. 38Persona tuning: iterate on system prompt + memories from real stream footage.
  6. 39Cost & latency optimization: cache, stream tokens→TTS, cheaper models for low-stakes lines.
Deliverable: a polished, robust, entertaining autonomous streamer.
